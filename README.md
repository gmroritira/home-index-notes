# home-index-notes

一个用于归档和发布独立 HTML 页面的仓库。不针对任何特定域名或网站，仅作为静态 HTML 资料存储与展示空间。

## 项目简介

本仓库存储多个独立的 HTML 页面，每个页面均为自包含的离线归档文件。页面内容可能涵盖笔记、工具页面、信息索引等不同类型，彼此之间无固定关联。仓库本身不提供索引或导航功能，仅作为归档与发布的载体。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 默认首页（可选）
└── pages/             # 独立 HTML 页面归档目录
    ├── page-01.html
    ├── page-02.html
    └── ...
```

- `index.html`：可选的默认首页，用于展示仓库概览或跳转链接。
- `pages/`：存放所有独立 HTML 页面的目录，每个页面为一个单独的文件。

## 页面归档说明

- 所有页面均为标准的 HTML 文件，可直接通过浏览器打开。
- 页面资源（如 CSS、JavaScript、图片等）建议内联或使用相对路径引用，以保持归档的独立性。
- 页面命名建议使用有意义的英文或拼音，避免特殊字符和空格。
- 如需添加新页面，请直接放入 `pages/` 目录，并在必要时更新 `index.html` 中的链接。

## 维护说明

- 本仓库不设自动构建或部署流程，所有页面均为静态文件，直接通过 GitHub Pages 或其他静态托管服务发布。
- 更新页面时请直接修改对应文件，或新增文件至 `pages/` 目录。
- 删除页面时请同时清理 `index.html` 中的相关链接（如有）。
- 建议保持页面数量适中，避免目录过于庞杂。

## 其他

- 本仓库仅作资料归档用途，不保证页面内容的时效性或准确性。
- 如使用 GitHub Pages 发布，请确保仓库设置中已启用 Pages 功能，并选择正确的源分支与目录。
